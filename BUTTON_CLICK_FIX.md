# Button Click Handler Fix

## Problem

Buttons, arrows (collapse/expand chevrons), close buttons (X), and other interactive elements in the SWAT+ Dataset Selector webview were not responding to clicks.

## Root Cause

The event listeners were only attached in a `DOMContentLoaded` event handler. When the webview HTML was regenerated by calling `_updateWebview()` (which happens when navigating directories, filtering files, etc.), the entire HTML was replaced. However, since the page wasn't being reloaded in the traditional sense, the `DOMContentLoaded` event did NOT fire again, so the event listeners were not re-attached to the new DOM elements.

This created a situation where:
1. Initial page load → DOMContentLoaded fires → Event listeners attached ✓
2. User navigates directories → _updateWebview() called → HTML replaced → DOMContentLoaded doesn't fire → Event listeners lost ✗
3. Buttons no longer respond to clicks

## Solution

The fix refactors the webview script to:

1. **Extract event listener setup into a reusable function** (`initializeEventHandlers()`)
   - All button, click, drag-drop, and other event listeners are now in this function
   - This allows the listeners to be re-attached on demand

2. **Create an API initialization function** (`initializeVsCodeApi()`)
   - Sets up the `vscode` API and `swatHost` wrapper object
   - Only initializes once (uses flags to prevent re-initialization)

3. **Attach listeners at multiple points:**
   - If DOM is still loading: Wait for `DOMContentLoaded` then initialize
   - If DOM is already loaded: Initialize immediately
   - After script load completion: Use `requestAnimationFrame` to re-initialize handlers as a fallback

4. **Declare `vscode` and `swatHost` at the IIFE scope**
   - Makes them accessible to all functions
   - Allows safe re-initialization without creating duplicates

## Code Changes

**File:** `src/swatWebviewProvider.ts`

**Key changes in the `<script>` section:**

```typescript
// BEFORE: Only DOMContentLoaded handler
document.addEventListener('DOMContentLoaded', () => {
    const vscode = acquireVsCodeApi();
    const swatHost = { ... };
    // All event listeners here
});

// AFTER: Reusable functions with multiple initialization points
let vscode;
let swatHost;

function initializeVsCodeApi() { ... }
function initializeEventHandlers() { ... }

// Initialize at multiple points
initializeVsCodeApi();
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', initializeEventHandlers);
} else {
    initializeEventHandlers();
}
requestAnimationFrame(() => initializeEventHandlers());
```

✓ All buttons now respond immediately when clicked
✓ Collapse/expand arrows work after navigation
✓ Close buttons (X) respond after HTML updates  
✓ All interactive elements maintain functionality after webview updates
✓ Drag-and-drop continues to work
✓ Filter checkboxes remain responsive
